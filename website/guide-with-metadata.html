Recipe D · 04 · ~7 min · beginner

Attach sample metadata to records.

.with_metadata(**fields) stamps every output record with arbitrary user fields — donor IDs, sample IDs, sequencing batches, timepoints, anything. The AIRR-standard fields sample_id, donor, and cell_id propagate as proper AIRR columns; everything else lands as user-extension columns alongside them. Mostly useful for multi-sample / multi-donor benchmarks where you need to keep panels distinguishable after they're stacked.

PART 01

AIRR-standard fields and arbitrary fields.

with_metadata takes any keyword arguments. Three of them — sample_id, donor, cell_id — are AIRR-spec column names and will be recognised by downstream AIRR-aware tools. Everything else lands as a user-extension column with the same name you passed.

AIRR-standard fields
  • sample_idthe AIRR column for distinguishing samples within a study
  • donorthe AIRR column for the source individual
  • cell_idthe AIRR column for single-cell data
Arbitrary user fields
  • protocol"MiSeq-v3" — sequencer/protocol tag
  • timepoint"day_14" — longitudinal label
  • replicate3 — replicate index, any JSON-serialisable type
  • anything you wantcolumn name = keyword name, value = the value
Stamp one panel with both standard + custom fields
import GenAIRR as ga

result = (
    ga.Experiment.on("human_igh")
      .with_metadata(
          sample_id="S001",         # AIRR standard
          donor="D-01",             # AIRR standard
          protocol="MiSeq-v3",       # user extension
          timepoint="day_14",       # user extension
      )
      .recombine()
      .mutate(model="s5f", count=(5, 25))
      .run_records(n=100, seed=42)
)

rec = result.records[0]
assert rec["sample_id"] == "S001"
assert rec["donor"]     == "D-01"
assert rec["protocol"]  == "MiSeq-v3"
assert rec["timepoint"] == "day_14"
PART 02

Multi-sample / multi-donor benchmarks.

The typical use case: simulate several donors or batches and stack them into one panel that an aligner sees as a single AIRR TSV. Loop over your donor list, stamp each run with its own donor tag, concatenate at the dataframe layer.

Three donors, one stacked panel
import GenAIRR as ga
import pandas as pd

panels = []
for donor in ("D001", "D002", "D003"):
    p = (
        ga.Experiment.on("human_igh")
          .with_metadata(donor=donor, sample_id=f"{donor}-S1")
          .recombine()
          .mutate(model="s5f", count=(5, 25))
          .run_records(n=5_000, seed=42, expose_provenance=True)
          .to_dataframe()
    )
    panels.append(p)

stacked = pd.concat(panels, ignore_index=True)
# 15,000 rows, distinguishable by stacked["donor"]

stacked.to_csv("multidonor_panel.tsv", sep="\t", index=False)

Re-using the same seed=42 across donors is intentional — V/D/J distributions stay comparable across donors. Vary the seed if you want each donor's draws to be independent.

PART 03

Single-cell shape: cell_id + clonal fork.

cell_id is AIRR's column for tying together heavy/light pairs or sister receptors from the same cell. Pair it with .with_clonal_structure(...) to stamp a single cell_id over an entire clonal lineage, useful as a synthetic single-cell panel.

cell_id propagates across clonal descendants
result = (
    ga.Experiment.on("human_igh")
      .with_metadata(sample_id="PBMC-batch-A", cell_id="cell_0001")
      .recombine()
      .with_clonal_structure(n_clones=1, size=12)   # 12 sister receptors
      .mutate(model="s5f", count=(5, 25))
      .run_records(seed=42)
)

# every record carries the same cell_id and sample_id
assert all(r["cell_id"]   == "cell_0001"    for r in result.records)
assert all(r["sample_id"] == "PBMC-batch-A" for r in result.records)

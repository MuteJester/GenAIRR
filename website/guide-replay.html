Recipe C · 03 · ~10 min · advanced

Replay a simulation with one knob changed.

Pin V/D/J via .using(...), fix the seed, and the engine produces a bit-identical recombination on every run. Vary one downstream knob — mutation model, mutation count, a corruption parameter — and any difference between the two outputs is attributable to that knob alone. No upstream RNG drift, no annotation noise, no other moving parts.

01 Pick a V/D/J anchor from a baseline run or refdata directly
02 Lock with .using(...) pins V, D, J to specific alleles
03 Vary one downstream knob same seed everywhere else
PART 01

Why this is single-variable isolation.

The engine's RNG is deterministic and the .using(...) allele lock pins each segment's candidate distribution to exactly one element. So at fixed seed, V allele selection, D allele selection, J allele selection, V/D/J trims, NP1 length, NP2 length, and the NP base draws all come out bit-identical across runs. The only thing that differs is whichever pass you swapped — the change is causally isolated to that pass.

What stays bit-identical (same lock, same seed)
  • V · D · J allele namesthe locked triple, by construction
  • V/D/J trim lengthsdeterministic from seed once alleles are pinned
  • NP1 · NP2 length and basesdrawn before any downstream pass, identical across runs
  • sequence_idindexed deterministically
What you can vary
  • Mutation models5f vs uniform with the same target count
  • Mutation count or rateparameter sweep against fixed germline
  • Corruption shapeadd or drop corrupt_* passes individually
  • Productive contracttoggle respect=ga.productive() on/off
PART 02

The lock-and-replay pattern.

Pick a V/D/J triple to anchor on (often by running the baseline once and reading truth_v_call off the record you want to study). Build a small panel function that hard-codes the lock and the seed; call it with whatever knob you're varying.

S5F vs uniform on the same recombination
import GenAIRR as ga

# 1. baseline run — read one record to discover a V/D/J triple to anchor on
baseline = (
    ga.Experiment.on("human_igh").recombine()
      .run_records(n=1, seed=42, expose_provenance=True)
).records[0]
V, D, J = baseline["truth_v_call"], baseline["truth_d_call"], baseline["truth_j_call"]

# 2. panel function — lock V/D/J, fix the seed, vary one knob
def panel(model):
    return (
        ga.Experiment.on("human_igh")
          .using(v=V, d=D, j=J)               # <-- the lock
          .recombine()
          .mutate(model=model, count=(15, 15))
          .run_records(n=10, seed=42, expose_provenance=True)
    ).records

s5f = panel("s5f")
uni = panel("uniform")

# 3. confirm the lock held — V/D/J identical, sequences differ (SHM only)
assert all(a["truth_v_call"] == b["truth_v_call"] == V for a, b in zip(s5f, uni))
assert all(a["sequence"] != b["sequence"]                for a, b in zip(s5f, uni))
PART 03

Sweeping a parameter.

The same lock-and-replay shape drives a one-knob parameter sweep — fixed V/D/J, varying mutation count, plot the ablation curve. The kind of figure that goes straight into a paper.

SHM count sweep on a single recombination
# V/D/J locked above; sweep mutation count, all 5 panels share the recombination
records = []
for mu in [5, 10, 15, 20, 25]:
    rec = (
        ga.Experiment.on("human_igh")
          .using(v=V, d=D, j=J)
          .recombine()
          .mutate(model="s5f", count=(mu, mu))
          .run_records(n=1, seed=100, expose_provenance=True)
    ).records[0]
    records.append({"mu": mu, "v_identity": rec["v_identity"]})

# v_identity drops monotonically with mu — clean ablation curve.
# for plotting: pandas.DataFrame(records).plot(x="mu", y="v_identity")
PART 04

Inspecting an intermediate state.

If you want to read the IR between two passes — say, to compare the pre-mutation germline against the post-mutation sequence on the same record — use Outcome.revision_after("pass_name") to retrieve the IR snapshot taken immediately after that pass. This is for introspection, not for re-running a different downstream from it (the engine doesn't currently support feeding a snapshot back into a new PassPlan; the lock-and-replay pattern above is the supported way to compare conditions).

Read the post-recombination IR for a single record
out = (
    ga.Experiment.on("human_igh")
      .recombine()
      .mutate(model="s5f", count=(15, 15))
      .run(n=1, seed=42)
)[0]

# list the pass names you can snapshot at
print(out.pass_names())
# ['sample_allele.v', 'sample_allele.d', 'sample_allele.j',
#  'trim.v_3', 'trim.d_5', 'trim.d_3', 'trim.j_5',
#  'assemble.v', 'generate_np.np1', 'assemble.d',
#  'generate_np.np2', 'assemble.j', 'mutate.s5f']

# snapshot the IR immediately after the last recombination pass
sim_after_recombine = out.revision_after("assemble.j")
# sim_after_recombine is the pre-mutation Simulation — inspect bases, regions, etc.

Recipe C · 04 · ~10 min · intermediate

Introspect the trace.

Every random draw the engine makes is recorded in the trace under a stable, addressable name: sample_allele.v, trim.j_5, np.np1.bases[3], mutate.s5f.pos[42]. Use Outcome.trace() to retrieve that record for any record in a batch and ask exactly what the engine sampled — by name, by prefix, or by counting how many draws a pass made.

PART 01

Three queries cover most use cases.

trace.find(address) for a known exact name, trace.prefix_query(prefix) for an enumerated subset, trace.prefix_count(prefix) when you just want to know how many. Returns are ChoiceRecord objects with two fields: address and value.

The three accessors
  • trace.find(addr)returns one ChoiceRecord (or None) — for a single exactly-named draw
  • trace.prefix_query(prefix)iterates ChoiceRecords whose address starts with prefix
  • trace.prefix_count(prefix)returns an int — how many records match this prefix
Address scheme
  • sample_allele.v / .d / .jthe allele draws (value = numeric AlleleId)
  • trim.v_3 / d_5 / d_3 / j_5per-segment trim lengths
  • np.np1.length · np.np2.lengthNP region lengths
  • np.np1.bases[k] · np.np2.bases[k]each NP base draw, indexed
  • mutate.s5f.pos[k]each SHM substitution position (when mutate was run)
PART 02

Reading one record's trace end-to-end.

Get an Outcome from .run(...), call .trace() on it, and query whatever you need. Below: read the V allele the engine picked, the NP1 length and bases it drew, and the count of SHM substitutions it made.

Find · prefix_query · prefix_count, on one record
import GenAIRR as ga

out = (
    ga.Experiment.on("human_igh")
      .recombine()
      .mutate(model="s5f", count=(5, 25))
      .run(n=1, seed=42)
)[0]

trace = out.trace()

# 1. find — one named draw, returns ChoiceRecord with .address and .value
v_id     = trace.find("sample_allele.v").value     # int AlleleId — 146
np1_len  = trace.find("np.np1.length").value       # int — 13
v3_trim  = trace.find("trim.v_3").value            # int — 0

# 2. prefix_query — iterate every record under a prefix
for rec in trace.prefix_query("np.np1.bases"):
    print(rec.address, rec.value)
# np.np1.bases[0] b'C'
# np.np1.bases[1] b'G'
# np.np1.bases[2] b'A'  ... (13 bases total)

# 3. prefix_count — quick total
n_shm = trace.prefix_count("mutate.s5f")            # int — 35

sample_allele.v returns the integer AlleleId the engine drew, not the allele name. Map it back via refdata.v_allele(id).name — see PART 04 for the full pattern.

PART 03

Counting draws per pass — a debugging tool.

prefix_count is the fastest way to verify "did this pass run as I expected". Use it to confirm a contract didn't reject a draw, an indel pass actually fired, or a mutation count is in the range you asked for.

Audit how many draws each pass made
out = (
    ga.Experiment.on("human_igh")
      .recombine()
      .mutate(model="s5f", count=(5, 25))
      .corrupt_5prime_loss(length=(5, 30))
      .run(n=1, seed=42)
)[0]

trace = out.trace()
for prefix in ("sample_allele", "trim", "np", "mutate"):
    print(f"{prefix}: {trace.prefix_count(prefix)} draws")
# sample_allele: 3      — V + D + J alleles drawn
# trim:          4      — V_3, D_5, D_3, J_5 trim lengths
# np:            15     — np1.length + np2.length + bases
# mutate:        35     — every SHM substitution recorded

Not every pass writes to the trace. Corruption passes that don't make per-base random choices (some end-loss configurations, for instance) record nothing. The trace is the inventory of sampled decisions, not a per-pass execution log.

PART 04

Mapping AlleleId back to a name.

sample_allele.{v,d,j} values are integer AlleleIds — efficient handles into the reference pool. Resolve them to names via the same RefDataConfig the experiment was built on.

AlleleId → allele name
import GenAIRR as ga

exp = ga.Experiment.on("human_igh").recombine()
out = exp.run(n=1, seed=42)[0]

refdata = exp.refdata                            # the RefDataConfig
v_id = out.trace().find("sample_allele.v").value
v_name = refdata.v_allele(v_id).name
print(v_name)                                # 'IGHVF10-G38*04'

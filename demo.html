    Live demo¶

    A five-minute tour of GenAIRR. Every code block on this page was executed against the released GenAIRR==2.2.0 on this machine; the outputs shown are the actual ones the snippets produced. Copy, paste, run.

    Why GenAIRR¶

    Most repertoire benchmarks compare an aligner against a real-data oracle (another aligner). GenAIRR inverts the setup: the engine emits AIRR records with their ground truth baked in by construction. Every V/D/J call, every junction, every counter is derivable from the persistent intermediate representation, not inferred. That means:

    • Aligner benchmarks have a true ground truth, not a stronger-aligner proxy.
    • Null models are biologically grounded, with controllable productive fraction, SHM rate, V-gene usage, clonal structure.
    • Every run is byte-stable across machines and platforms, replayable from a 13 KB trace file with no side data.

    The next three sections show exactly how.

    1. Five lines, 1,000 annotated records¶

    import GenAIRR as ga

result = (
    ga.Experiment.on("HUMAN_IGH_OGRDB")
      .recombine()
      .productive_only()
      .mutate(model="s5f", rate=0.03)
      .run_records(n=1000, seed=42, expose_provenance=True)
)
df = result.to_dataframe()

    That ran in 2.0 seconds. The result is a pandas-ready DataFrame of 1,000 records and 102 AIRR columns, every field derived from the engine's internal state, not inferred. Here is the first record:

    Field Value
    sequence_id seq0
    v_call IGHVF10-G38*04
    truth_v_call IGHVF10-G38*04
    d_call IGHD2-15*01
    truth_d_call IGHD2-15*01
    j_call IGHJ2*01
    truth_j_call IGHJ2*01
    junction_aa CVKDDGNRGYCSGDSCYGHCCALDYWYFDLW
    productive True
    n_mutations 10 (V: 7, D: 1, J: 1, NP: 1)

    The truth_* columns record the alleles the engine committed at recombination time. The non-truth columns are what the projection would emit if it scored the (potentially mutated, potentially corrupted) sequence against the catalogue. On a heavy-SHM record the two diverge, which is exactly the signal an aligner benchmark should be measuring.

    2. Constraint-aware sampling, not reject-and-retry¶

    Most simulators build a sequence, check whether it is productive, discard if not. GenAIRR registers a contract set at compile time and the relevant sampling sites consult it to prune their support before each draw.

    The observable effect:

    unconstrained = ga.Experiment.on("HUMAN_IGH_OGRDB").recombine().run_records(n=1000, seed=42)
constrained  = ga.Experiment.on("HUMAN_IGH_OGRDB").recombine().productive_only().run_records(n=1000, seed=42)

print(f"Without productive_only(): {sum(1 for r in unconstrained if r['productive']) / len(unconstrained):.1%} productive")
print(f"With    productive_only(): {sum(1 for r in constrained   if r['productive']) / len(constrained):.1%} productive")

    Without productive_only(): 18.5% productive
With    productive_only(): 100.0% productive

    Four predicates back the bundle:

    Predicate What it enforces Fires at
    AnchorPreserved(V) V Cysteine codon survives the 3' V trim V-trim sampling
    AnchorPreserved(J) J Trp / Phe codon survives the 5' J trim J-trim sampling
    ProductiveJunctionFrame Junction length divisible by 3 NP-region length sampling
    NoStopCodonInJunction No TAA / TAG / TGA in the assembled junction NP base-draw sampling

    The constraint set is registered at compile() time, not consulted per record. Sampling sites that the contract is silent about run unconstrained at full speed.

    3. Byte-stable replay + structured validation¶

    This is the differentiator. GenAIRR records every sampled choice in a trace, gates replay on three independent signatures, and re-derives every output field from the underlying outcome to catch drift.

  C --> D["byte-identical Outcome"]

    Live:

    exp = (
    ga.Experiment.on("HUMAN_IGH_OGRDB")
      .recombine().productive_only()
      .mutate(model="s5f", rate=0.03)
)
compiled = exp.compile()

# 1. Run + save trace
outcome    = compiled.simulator.run(seed=42)
trace_file = compiled.simulator.trace_file_from(outcome, seed=42)
trace_file.write_to("demo.trace.json")           # 13 KB on disk

# 2. Replay — byte-identical Outcome
from GenAIRR._engine import TraceFile
tf       = TraceFile.read_from("demo.trace.json")
replayed = compiled.simulator.replay_from_trace_file(tf, strict=False)

# 3. Validate every field
from GenAIRR import SimulationResult, dataconfig_to_refdata
refdata = dataconfig_to_refdata(ga.HUMAN_IGH_OGRDB)
report  = SimulationResult.from_outcomes([outcome, replayed], refdata).validate_records(refdata)

print(f"ok = {report.ok}, issues = {report.summary()}")

    ok = True, issues = {}

    The replayed record is byte-identical to the original on every sampled-from-IR field:

    Field Match
    sequence True
    v_call / d_call / j_call True
    junction_aa / junction_length True
    n_mutations (and per-segment partition) True

    The trace file itself is small (13 KB for one outcome) and carries the three replay gates as top-level keys:

    Field Purpose
    pass_plan_signature Hash of every DSL method + kwarg. A pipeline change flips this.
    refdata_signature Hash of cartridge identity. A different cartridge flips this.
    refdata_content_hash Hash of cartridge content. A re-estimated empirical plane flips this.

    All three must match for replay to consume the trace. That's tighter than a seed, and it survives engine refactors that leave the public surface intact.

    validate_records() runs an independent re-derivation of every AIRR field from the underlying events and asserts agreement. The 13 KB and the validator together are the byte-stability guarantee.

    4. And there's more¶

    Two capabilities the demo doesn't deep-dive but worth mentioning:

    Clonal families — fork an ancestor, let SHM diverge per descendant:

    result = (
    ga.Experiment.on("HUMAN_IGH_OGRDB")
      .recombine().productive_only()
      .expand_clones(n_clones=10, per_clone=50)
      .mutate(model="s5f", rate=0.02)
      .run_records(seed=42)
)
# 500 records, 10 clones, 50 per clone — same V/D/J in each clone,
# divergent SHM counts: e.g. clone 0 descendants n_mutations = 4, 14, 8, ...

    Reference cartridges from your own AIRR data — five estimators turn a rearrangement table into a custom cartridge with empirical allele usage, trim distributions, NP length / base / P models. See Estimate cartridge models.
